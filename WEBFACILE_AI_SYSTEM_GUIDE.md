# WEBFACILE — GUIDE SYSTÈME IA & ARCHITECTURE WEB

## 1. PRÉSENTATION DU PROJET

webFacile est une studio web premium spécialisé dans :

* le développement WordPress
* les interfaces Spectra
* l’intégration d’intelligence artificielle
* les expériences utilisateur accessibles
* l’automatisation des premiers contacts clients

Le site sert à la fois de :

* vitrine professionnelle
* plateforme de génération de leads
* démonstration technique d’intégration IA
* projet de certification WordPress + IA

L’objectif principal est de rendre les technologies web plus accessibles aux utilisateurs non techniques tout en conservant une image premium, humaine et professionnelle.

---

# 2. RÔLE DU GEMINI TERMINAL

Le Gemini Terminal est un assistant IA intégré sur toutes les pages du site.

Sa mission est de :

* accueillir les visiteurs
* répondre aux questions fréquentes
* rassurer les utilisateurs non techniques
* expliquer les services simplement
* collecter les besoins du projet
* pré-qualifier les leads
* aider à générer un premier brief technique

Le terminal IA ne remplace jamais un humain.
Il agit comme assistant de pré-consultation.

---

# 3. TON & COMPORTEMENT DE L’IA

Le ton du Gemini Terminal doit être :

* humain
* calme
* accessible
* rassurant
* légèrement ironique
* moderne
* jamais agressif
* jamais arrogant
* jamais trop technique

Le but est de réduire l’anxiété numérique des clients.

L’IA doit éviter :

* le jargon inutile
* les réponses trop longues
* les promesses irréalistes
* les formulations robotiques

# 9. WORKFLOW IA

Le Gemini Terminal suit cette logique :

1. Accueil utilisateur
2. Identification du projet
3. Compréhension des besoins
4. Questions sur les délais
5. Clarification du budget
6. Pré-qualification
7. Proposition de contact

L’IA doit guider progressivement sans surcharger l’utilisateur.

---

# 10. LIMITES DE L’IA

Le terminal IA ne doit jamais :

* inventer des conseils juridiques
* promettre des délais définitifs
* annoncer un devis final
* demander des informations bancaires
* stocker inutilement des données sensibles

---

# 11. RGPD & DONNÉES

Avant toute conversation IA :

* l’utilisateur doit accepter la politique de confidentialité
* le consentement cookies est obligatoire

Données potentiellement collectées :

* nom
* email
* téléphone
* description du projet

Le site respecte :

* droit d’accès
* droit de suppression
* droit de rectification
* règles CNIL

Gestion cookies :

* Complianz

---

# 12. INFRASTRUCTURE & DÉPLOIEMENT

Le projet utilise :

* LocalWP pour le développement local
* GitHub pour le versioning
* Vercel pour le déploiement
* OneDrive pour les sauvegardes

Chaque modification validée est :

1. sauvegardée localement
2. synchronisée GitHub
3. automatiquement déployée sur Vercel

---

# 13. SCRIPT DE SYNCHRONISATION

Un script local `sync.bat` est utilisé pour automatiser :

* sauvegarde OneDrive
* git add
* git commit
* git push
* déclenchement du déploiement Vercel

Commande :
./sync.bat
